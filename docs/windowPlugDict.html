補足

なお、サブルーチンと関数で実行される処理は、メニューや ウインドウの登録 にあるボタンと同じである。

よって、保存の処理がファイルに登録される事も同じである。

ウインドウを移動やサイズの変更をする処理をした時も、一度だけ保存をして 元に戻す で戻せる様にする仕様も同じである。

Win_Plug_O_ から始まるものは設定用の変数である。

TRR の O_ から始まる設定用の変数と同じ様なものである。

Win_Plug_SetO_ から始まるものは設定用の変数のセッターの関数である。

TRR の SetO_ から始まる関数と同じ様なものである。

Win_Plug_O_ から始まる変数は代入して使う設定の変数だが、

Win_Plug_O_menuPosX = 100

として設定するのではなく、

Win_Plug_SetO_menuPosX( 100 )

を使って代入して設定する事を薦めたい。

Win_Plug_B_ から始まるものは変数である。

TRR の B_ から始まる TRR の変数と同じ様なものである。

値を代入しないで欲しい変数である。

ただのグローバル変数なので、代入すると普通に中の値が変更されてしまう事になる。

主に、 windowPlug の Gui の番号や、Gui のウインドウタイトルが格納された変数がほとんどである。

Win_Plug_Menu から始まるものは、変数やサブルーチンでもなく、メニュー名 ( カスタムメニュー名と呼ばれているもの ) である。

TRR では D_menu から始まるものは、メニュー名になっているのだが、それと同じ様なものである。

使い方として、

Menu, Win_Plug_MenuMain , Show , %Win_Plug_O_menuPosX% , %Win_Plug_O_menuPosY%

といった様な使い方をする。

Menu の AHKコマンド を使う時に指定するメニュー名の事である。

Win_Plug_Sub から始まるものはサブルーチンである。

TRR の E_ から始まるサブルーチンと同じ様なものである。

windowPlug には、このサブルーチンが数多くある。

なぜなら、 windowPlug のメニューの項目のものを全てこのサブルーチンとして用意しているからである。

逆に言えば、メニューにあるものしか用意していない。

なので windowPlug を知っていれば動作に予想がつくサブルーチンばかりだといえる。

Win_Plug_ から始まるが、以上に挙げた例のどれでもなく、途中から ( がつき、 ) で終わるものは関数である。

windowPlug には、この関数も数多くある。

とはいえ、 Win_Plug_Sub から始まるサブルーチンと同じ様な動作をするものも多い。

サブルーチンと違い引数を指定する事で細かく指定しながら使える様に用意しているものである。

むしろ、めんどうなのでサブルーチンと違って全部用意していないような感じである。

もう少し細かい名前のルール

これを知っていれば、名前を見ただけで予想がつきやすくなると思われる。

と言っても、 windowPlug のメニューを見て確認した方がもっと予想がつく様になると思うのでそちらを知る方が大事だと思われる。

Win_Plug_SubMenu が初めにつくものはメニューを表示するためのサブルーチンである。

ちなみに、

Gosub , Win_Plug_SubMenuMain

このスクリプトで windowPlug のトップのメニューが表示する。

Win_Plug_SubGroup が初めにつくものは グループ 関係の操作を表している。

メニューの、 グループ 以下の項目がこれに該当するということである。

グループ の 追加 の項目は

Gosub , Win_Plug_SubGroupAdd

これを使うと同じ事ができる。

切り替え は

Gosub , Win_Plug_SubGroupActive

グループに関する事は、 アイテム にするか、 常時使用のキー にするかをしないと特に使う機会はないものと思われる。

Win_Plug_SubShow が初めにつくものは windowPlug の Gui のウインドウを表示させるものである。

Gosub , Win_Plug_SubShowEntryWinPosSize

これを使うと、 ウインドウの登録 のウインドウを表示する。

ちなみに今のは、

Gui , %Win_Plug_B_GuiNumEntryWin%:Show

これを使っても大して変わらない。

Win_Plug_SubPos が初めにつくものは、
移動 のメニュー以下の項目を表している。

Win_Plug_SubSize が初めにつくものは、
サイズ のメニュー以下の項目を表している。

少し不親切で分かりにくいが、 サイズと移動 のメニュー以下の項目は PosSize とは書かれていない。

代わりに U 、 D 、 L 、 R の文字が書かれている。

  • U は Up で上、
  • D は Down で下、
  • L は Left で左、
  • R は Right で右を表している。

例をだすと、

Gosub , Win_Plug_SubLR3bunL

これは、 サイズと移動 → ３分割 → 左右の左 この項目と同じものを意味する。

名前に Backup があるものは、 保存 と同じものを意味している。

名前に Before があるものは、 元に戻す と同じものを意味している。

サブルーチンの名前の中にあるものの意味を長々と説明したが、以上が全てである。

以上の意味が分かれば、名前を見ただけで何を意味しているかが予想がつくと思われる。

よって、このページを使って各サブルーチンの仕様をいちいち調べる必要はほとんど無いのではないかと思われる。

関数の名前のルール

次は関数の名前のルールを書いておく。

引数を指定しながら使えた方がいいものはサブルーチンではなく関数として作っている。

一応、サブルーチンの時と共通しているので、繰り返しの様な説明は多くなる。

名前に pos とつくものは、 移動 に関するもの。

Win_Plug_posChange(100, 200)

この関数はウインドウの位置を変える関数である。

名前に size とつくものは、 サイズ に関するもの。

Win_Plug_sizeChange(600 , 500)

この関数はウインドウのサイズを変える関数である。

名前に posSize とつくものは、 サイズと移動 に関するもの。

Win_Plug_posSizeChange(100 , 200 , 600 , 500)

この関数はウインドウのサイズと位置を変える関数である。

名前に backup があるものは、 保存 と同じものを意味している。

名前に before があるものは、 元に戻す と同じものを意味している。

名前に only があるものは、片方だけを指定するもの。

Win_Plug_posChangeOnlyX( 100 )

これは、X の位置だけを変えて Y の位置はそのままにしておく事になる。

名前に entryList があるものは、 ウインドウの登録 のリストを使う事を表す。

Win_Plug_posChangeFromEntryList( 1 )

これは、 ウインドウの登録 の位置のリストの set1 に登録してある内容を使って位置を変える処理になる。

名前に entryListPlusAssign があるものは少し複雑だが、 ウインドウの登録 のリストを基本的に使い、さらに引数で指定した事を上書きして使う関数である。

ここでは、

Win_Plug_posChangeFromEntryListPlusAssign( listNumber, posX="", posY="", winTitle="", absoleStr="" )

この関数を例にして説明してみる。

[ 例 ]

Win_Plug_posChangeFromEntryListPlusAssign( 1 , "" , 50 )

これは、 ウインドウの登録 の位置のリスト ( 今回は set1 ) に登録してある内容を基本的に使って位置を変えるが、引数に使ったものを優先して位置を決める処理である。

  • 引数1 はリストの番号の事なので、 1 の指定は set1 を意味し、

  • 引数2 は X の位置を意味しているが、 "" というカラの文字なので set1 に登録してある内容を使用し、

  • 引数3 は Y の位置を意味していて、 50 を指定しているので、縦位置が 50 の位置になる。

つまり、 set1 に登録している内容からは、 X の位置を読みだした値を使って、 Y の位置は 50 にした処理を行うという事である。

実は、 引数4 の winTitle と
引数5 の absoleStr のとこも set1 に登録している内容から使われる事になる。

名前に save があるものは、 ウインドウの登録 のウインドウの 保存 ボタンと同じ様な事をするものである。

引数に登録する内容を記入するが、登録する内容を全て記入するので引数が多い。

名前に saveWinTitle があるものは、上のと同じ様に ウインドウの登録 のウインドウの 保存 ボタンと同じ様な事をするものである。

違いは、今表示しているウインドウの中からウインドウタイトルを指定して、それの情報を保存する。

ここでは、

Win_Plug_sizeChangeSaveWinTitle( listNumber, getWinTitle="A", useWinTitle="A", listName="" )

この関数を例にして説明してみる。

[ 例 ]

Win_Plug_sizeChangeSaveWinTitle( 1 )

これは、最前面のウインドウのサイズを set1 に保存する処理である。

この関数の 「引数2」 の getWinTitle は、省略すると "A" を指定した事と同じ事になるので、最前面のウインドウの情報を取得する。

以下に [ 例 ] の引数の意味を細かく説明する。

  • 引数1 はリストの番号の事なので、 1 の指定は set1 を意味し、

  • 引数2 は取得するウインドウのウインドウタイトルの指定になるが、指定を省略しているので "A" を指定している事と同じになる。

    AHK のルールと同じで "A" の指定は、最前面のウインドウを意味する。

  • 引数3 は読みだして実行する際、どのウインドウに対して使うかの指定である。

    そのウインドウタイトルの指定である。

    "A" を指定している場合は、最前面のウインドウを対象に実行する。

  • 引数4 の listName は、 リスト名 のとこと同じである。

    カラの文字である "" を指定した場合は、 リスト名 を空欄にした場合と同じ。

先に書いた事だが、これらの関数は、 getWinTitle などの引数を省略した方が、最前面のウインドウを取得して、最前面のウインドウに対して実行するシンプルなものを作りやすい。

名前に load があるものは、 ウインドウの登録 のウインドウの 読み出し ボタンと同じ様な事をするものである。

引数にリストの番号を指定する。 set1 なら、 1 を指定する。

移動 の種類なら、位置の情報を set1 などから取得して位置の指定のウインドウ ( Gui ) を表示する。

名前に loadWinTitle があるものは、上のと同じ様に
サイズ の種類なら、サイズの指定のウインドウ ( Gui ) を表示する。

違いは、今表示しているウインドウの中からウインドウタイトルを指定して、それの情報を取得して表示する。

引数の名前のルール

引数の名前のルールを知っておけば、何を指定するための引数かが分かるので書いておく。

補足

引数に ="A" などとあった場合、その引数は省略できる引数を意味している。

="A" の場合は引数の指定が省略された場合、 "A" を指定したのと同じ事になる。

初期値みたいなものである。

もっとも、このルールは AutoHotkey の言語にあるルールと同じものである。

posX

横の位置。数字で指定する。

数字が大きいほど右に行く。

posY

縦の位置。数字で指定する。

数字が大きいほど下に行く。

width

横幅。数字で指定する。

height

高さ。数字で指定する。

winTitle

ウインドウタイトルの指定。

AHK のウインドウタイトルの指定と同じ。

"A" の指定なら、最前面のウインドウを指定している事になる。

補足

この windowPlug では、引数を省略すると "A" が自動的に記入された事になる関数が多い。

"" のカラの文字を記入しても、 "A" を指定した事と同じになる。

AHK のウインドウタイトルの指定と同じなので、
"ahk_class  [クラス名]" の様なクラス名の指定なども可能である。

getWinTitle

ウインドウタイトルの指定と同じだが、位置やサイズの情報を取得する時に使われる方のウインドウタイトル。

useWinTitle

ウインドウタイトルの指定と同じだが、実行する時の対象になる方のウインドウタイトル。

listNumber

ウインドウの登録 の Gui のリストの番号。

数字で指定する。

1 を指定した場合は、 set1 を指定した事になる。

absoleFlag

位置やサイズを指定する Gui の 絶対指定 と プラス指定 のスイッチ式のチェックボックスと同じものを意味している。

True の指定だったら 絶対指定 の扱いになる。

AHK では、 True は値が 1 と同じ扱いになる。
よって、 1 か 0 の指定でもよい。

absoleStr

これも、位置やサイズを指定する Gui の 絶対指定 と プラス指定 のスイッチ式のチェックボックスと同じものを意味している。

値の指定が文字列でもよく、

  • "on" か "off"
  • "True" か "False"
  • "1" か "0"

この様な指定の仕方でもいいもの。
大文字、小文字はどちらでもよい。

左が True で、右が False と同じ事を意味している。

absoleFlag の様に、
True か False 、
1 か 0 の指定でも別によい。

指定が "" の様にカラだったら、 True が指定された事と同じ事になる。

listName

ウインドウの登録 の Gui の リスト名 と同じ。

指定したら、リスト内の set1 などの文字が指定した文字になる。

文字を変えたとしても、 "" の様にカラの指定にすれば、 set1 などの文字に戻る。

ひたすらルールが書かれているが、説明する事はこれで以上である。

せっかくここまで説明しておいてなんですが、何度か書いている通りこれらを知ったところで大した事はできません。

所詮、私ブランボンが趣味で作った程度のものでしかありません。
