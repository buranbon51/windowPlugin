前書き

ウインドウの位置とサイズを変更する TRR のプラグイン（追加項目）の様なものを紹介して説明します。

とりあえず、名前を windowPlug と名付けています。

ここで説明する windowPlug は
ソースコード版の TextRunRun でのみ使用できるものです。

ソースコード版の TRR に追加する事で初めて使用できます。

追加の手順は後で説明します。

わざわざ紹介しておいて何ですが、そんなに大したものではありません。

ウインドウの位置とサイズを変更するものを登録して使っていける様にしたものです。

概要

基本的に以下の画像の左上にあるメニューを使っていくものです。

メニューを見ると分かると思いますが、

  • 移動
  • サイズ
  • サイズと移動
  • グループ

この 4 種類に大きく分かれています。

ウインドウに対してこれらの操作と登録をしていくものです。

とりあえず例として 移動 のメニューの

登録を表示 をたどると以下のウインドウが表示します。

このウインドウは、ウインドウの位置の情報を登録するリストといったものです。

これを見て分かるとおり、この windowPlug はウインドウの情報を登録しておいて、その後に使っていける様にするものです。

以上の事を見れば使い方と出来る事はほぼ予想がつくようなものだと思います。

とはいっても、使ってみなくてもここで全て把握できる様にするために以下にこまごまと説明をしていく事にします。

ウインドウの情報を登録する

先ほどあった、ウインドウの情報を登録するリストがあるウインドウを使います。

先ほどと違って、メニューの サイズと移動 と 登録を表示 を経由した場合で説明します。

このウインドウを ウインドウの登録 のウインドウと呼んでいきます。

そして、 最前面のを保存 のボタンを押したとします。

ウインドウの登録 のウインドウの一つ下のウインドウの、「位置と大きさ」の情報が数字で表示されているウインドウが表示します。

そして、この「位置と大きさ」の数字が表示されているウインドウがある時に、 ウインドウの登録 のウインドウの方の
保存 のボタンを押すと、「位置と大きさ」のウインドウの情報が登録されます。

ここの場合では、リストの set1 に登録された事になります。

補足

「位置と大きさ」のウインドウが表示されていなくても、 保存 のボタンは使用できます。
ただしその状態だと、どの情報が set1 に上書きされるかが分かりずらい事になります。

後は予想がつく事でしょうが、登録をした後はそれを今後に使っていける事になります。

ウインドウの登録 のウインドウの方の
読み出し のボタンを押すと「位置とサイズ」のウインドウが表示し、 set1 などにある情報が記入された状態になります。

「位置とサイズ」のウインドウの
On のボタンを押すと、その情報を一つ下のウインドウの「位置と大きさ」に対して変更させます。

実行 のボタンを押すと、 set1 などにある情報の通りに ウインドウの登録 のウインドウの、一つ下のウインドウの「位置と大きさ」を変更させます。

ソースコードへの追加の仕方

この windowPlug は
TextRunRun のソースコードに追加しないとそもそも使えません。

今になってですが、使える様にするための方法を説明します。

諸注意

この windowPlug は、 41 から 44 の Gui の番号を使用しています。

TRR の中で番号が同じ Gui をユーザーが既に使用している場合は注意してください。

同じ様なプラグインである mousePlug というのがありますが、それと同時に使う事は可能です。

windowPlug のダウンロード

GitHub のページからソースコードをダウンロードします。

windowPlug のソースコード

TRR のソースコードをダウンロードした方なら、 GitHub からそれをする方法も説明不要だと思います。

TRR のソースコードをダウンロードした方でないとそもそも windowPlug の追加はできません。

TextRunRunのWebサイト

TRR のソースコードの事については以下のページで説明しています。

windowPlug のフォルダをコピーする

windowPlug をダウンロードして解凍すると、以下のファイル構成になっていると思います。

この中の src の中にある、
windowPlug のフォルダを使います。

このフォルダだけを、 TRR のフォルダの中の

trrEvery\Plugins

この中にコピーしてください。

ソースコードを記入する

次は、ファイルの中に文字を記入します。

trrEvery\Plugins

このフォルダの中にある、
pluginsSubroutineAndFunction.ahk のファイルに以下を記入します。 

#Include %A_ScriptDir%\trrEvery\Plugins\windowPlug\useForPlug\windowPlugSubrouAndFunc.ahk

そして、同じフォルダ内にある
pluginsStartUpExecute.ahk のファイルに以下を記入します。 

#Include %A_ScriptDir%\trrEvery\Plugins\windowPlug\useForPlug\windowPlugStartUp.ahk

少し手間がかかりますが、以上で windowPlug を使える様にする準備は終了です。

TRR を起動すれば使える様になっているはずです。

使える様になったかどうかのテストとして

Gosub, Win_Plug_SubMenuMain

これを使ってみてください。

これを使うと、先ほど説明に挙げたメニューが表示するはずです。

左上にあるこのメニューが表示されれば正しくソースコードが追加されている事になります。

それと、うまくいったなら、このスクリプトを 常時使用のキー か アイテム にしておけば手軽に使える様になると思います。

位置とサイズのウインドウについて

使い方の説明に戻ります。

大体の予想はつく事でしょうが、このウインドウの事について説明しておきます。

X （横）と Y （縦）の位置

Width （横幅）と Height （高さ）の大きさ

これらを数値で指定します。

On のボタンを押すと、一つ下のウインドウに対して位置とサイズの変更をします。

絶対指定とプラス指定

絶対指定 と プラス指定 のスイッチタイプのチェック項目があります。

絶対指定 が普通の状態だと思ってください。

プラス指定 を選んでいる場合、例えば
Width が 100 なら、横幅が 100 の分だけ追加で大きくなります。

補足

プラス指定 を使う場合は指定した場所の他の箇所は全て 0 に指定して使った方が紛らわしくないと思います。

予想がついている事を言いますが、
位置の指定 の場所が、 X と Y に対して。

サイズの指定 の場所が、 Width と Height に対して影響をします。

ウインドウタイトル

一番下に ウインドウタイトル の入力欄があります。

ここに何も記入されていない場合は、この「位置とサイズ」のウインドウの一つ下のウインドウが対象になります。

ちなみに、 A のみが記入されている場合も同じです。

ここに何らかの文字が記入している場合は、そのウインドウタイトルのウインドウに対してのみ実行します。

当てはまる対象が複数ある場合は、対象になる一番上のウインドウ一つだけが対象です。

補足

TRR のウインドウタイトルの指定は、前方一致です。

諸注意

前方一致のルールを、ユーザーが意図的に変える事は可能ですが、変えた場合は テキスト上のキー が使用できなくなってしまいます。

基本的には、何も記入していない状態が使いやすいはずです。

メニューにある項目について

メニューにある項目についてある程度説明しておきます。

この windowPlug は、 移動 と サイズ と サイズと移動 に分かれています。

これらの違いは予想がつくものだと思います。これらのメニュー項目はほとんどが共通しています。

ここでは、 サイズと移動 のとこを例に使っていきます。

保存と元に戻す

メニューの下の方に
保存 と 元に戻す があります。

保存 を選ぶと、ウインドウの位置とサイズの情報が保存されます。

保存された後は 元に戻す を選ぶ事で保存していた情報の時の位置とサイズに戻します。

といっても、 この windowPlug を使ってウインドウの位置かサイズを変える場合は、この 保存 と同じ処理を先に行う様にしています。

なので、 保存 を使う事はあえてしなくてもよいと思います。

諸注意

この 保存 と 元に戻す の処理はウインドウタイトルを参考にして行います。

ソフトによっては、ウインドウがアクティブかどうかでウインドウタイトルが変わってしまうものがあったりします。

その場合は思うように動作しない可能性が高いです。

エディターのソフトに多いです。ウインドウがアクティブになるとウインドウタイトルがファイルパスになるものが多い様です。なのでエディターでは思うように 元に戻す が動作しないと思います。

補足 1

この 保存 の動作は、

%A_ScriptDir%\trrEvery\user\windowPlug\backupPosSize.ini

このファイルに保存されます。

このファイル内にウインドウタイトルごとに区別されて記録する感じです。

元に戻す を使った時は、その情報は削除されます。

このファイル内にあるそのウインドウタイトルの情報を削除する感じです。

補足 2

この windowPlug を使って位置かサイズを変える場合は 保存 と同じ処理を最初に行うと書きました。

その場合の 保存 の動作は、一度だけです。

同じウインドウタイトルの情報が保存されていた場合は 保存 をせず、情報の上書きをしません。

なので最初の 保存 だけが有効になります。

ただ、メニューから 保存 選んだ場合は上書きをする様になっています。

なので、明示的に今の状態を保存したい場合は 保存 をメニューから選んでください。

取得

メニュー内に 取得 というものがあります。

それを選ぶと

「位置とサイズ」のウインドウが表示し、アクティブウインドウの位置とサイズが入力欄に記入されます。

メニュー内にある プラス指定 を選ぶと、
「位置とサイズ」のウインドウが表示し

絶対指定 と プラス指定 のスイッチタイプのチェック項目が
プラス指定 の方にチェックが入った状態で表示します。

後、入力欄に 0 が記入されます。

メニュー内にある 数字で指定 を選んだ場合は、「位置とサイズ」のウインドウが表示するというだけです。

メニューで説明しておく事は大体はこれだけです。

何度も書いてしまってますが、 移動 と サイズ と サイズと移動 がありますが、これらのメニューはほぼ共通しているので違いは説明しなくても分かると思います。

補足

ちなみに、 移動 と サイズ にある、
保存 と 元に戻す は、
サイズと移動 にある
保存 と 元に戻す と何ら変わりはありません。

サイズ の方にある 保存 が、サイズだけを保存したりとかはしません。

少し残念かもしれませんが、現時点ではそうなっています。

必要な説明は大体以上だと思います。

この windowPlug は使っていけばすぐに分かると思いますのでこれ以上読まなくても普通に使う分には大丈夫だと思います。

よってこれ以降も windowPlug についての説明がいろいろ続きます。

これ以降は windowPlug の細かい仕様についての事になります。

ですので、さらにいろいろ知りたい方はこれ以降も読んでみてください。
